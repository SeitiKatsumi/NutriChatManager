# Overview

NutriChatBot is a comprehensive platform designed for nutritionists to streamline client management, enhance communication via WhatsApp, and leverage AI-powered chatbot services. It supports nutritionist registration, manages user and WhatsApp instances, and facilitates real-time messaging within a modern web application. The platform aims to optimize nutritionist workflows, improve client interaction through automated WhatsApp responses, and provide AI-driven insights for patient care and personalized meal planning.

# User Preferences

Preferred communication style: Simple, everyday language.

All phone numbers must be stored in Directus with Brazil country code "55" prefix:
- Format in database: `5511983283363` (digits only, including country code)
- Display format in UI: `+55 (11) 98328-3363` (Brazilian mask)
- Automatic normalization: 10-11 digit numbers get "55" prepended before saving

# System Architecture

## Frontend
Built with React 18 and TypeScript, utilizing shadcn/ui for design consistency and Wouter for routing. TanStack Query manages server state. The UI features a dark theme and a robust component library, with Vite ensuring fast development.

## Backend
Developed with Express.js and TypeScript, providing a RESTful API with a layered architecture including route handlers, a storage abstraction, and middleware for logging and error handling.

## Database
PostgreSQL is used with Drizzle ORM for type-safe database operations. Key entities include nutritionists, WhatsApp instances, and messages, linked by foreign keys and UUID primary keys.

## Authentication
Session-based authentication is implemented using `connect-pg-simple` for PostgreSQL-backed session storage and secure cookie handling.

## WhatsApp Integration
The platform directly integrates with Baileys (`@whiskeysockets/baileys`) for WhatsApp communication, replacing previous external dependencies. Each nutritionist's WhatsApp session is managed by `BaileysService`, with authentication states persisted across server restarts. Incoming messages trigger events for AI agent processing, ignoring groups and broadcast messages.

## AI Consultation & Patient History
AI-powered consultation analysis uses OpenAI, integrating with patient conversation history stored in a `whatsapp_messages` Directus collection. This enables quick insights, custom question answering, and personalized 24-hour meal plans generated by GPT-4o-mini, considering dietary needs and goals.

## Internal AI Agent System
Replaces external workflow automation with an in-app AI agent system for handling incoming WhatsApp messages.

**Key Components:**
- `server/whatsapp-message-handler.ts`: Dispatches Baileys events, identifies patients, routes to AI agents, and sends responses.
- `server/openai-service.ts`: Contains methods for `runAnamnesisAgent`, `extractPatientData`, `runFollowUpAgent`, and `analyzeFood`.

**AI Agents:**
- **Anamnesis Agent**: Conducts multi-step initial patient data collection, asking one question at a time and covering personal, health, dietary, lifestyle, and 24h food recall.
- **Follow-up Agent**: Provides personalized support to patients in the "Acompanhamento" stage based on their Directus profile.
- **Food Image Analysis**: Utilizes OpenAI Vision (gpt-4o-mini) to estimate calories and macros from food photos.

**Workflow:**
Incoming messages are processed by the handler, identified by nutritionist and patient, saved to `whatsapp_messages`, and routed to the appropriate AI agent. AI responses are saved and sent via Baileys. Anamnesis completion triggers data extraction, patient profile updates in Directus, and stage transition to "Acompanhamento".

**Patient Stage Cache:** A local cache (10-min TTL) prevents Directus delays, ensuring correct agent routing.

**Conversation Memory:** Utilizes Directus `whatsapp_messages` for persistent storage and an in-memory `conversationCache` (60 messages, 30-min TTL) for immediate context. `buildConversationHistory` merges and deduplicates data.

**Concurrency Protection:** `withPatientLock` serializes message processing per patient to prevent conflicts.

**WhatsApp JID Filtering:** Baileys message handler filters all non-user JID types at the source level using Baileys utilities: `isJidGroup` (groups), `isJidBroadcast` (broadcast lists), `isJidStatusBroadcast` (status), `isLidUser` (Linked Device IDs — internal WhatsApp identifiers that are NOT phone numbers), and `isJidNewsletter` (channels). Only messages from `@s.whatsapp.net` JIDs (real phone numbers) are processed. The message handler additionally validates phone number format (Brazilian: 12-13 digits starting with 55; international: 10-15 digits).

## Admin AI Configuration Panel
An admin panel at `/admin` allows managing AI agent configurations without code changes.

**Architecture:**
- `server/ai-config-store.ts`: PostgreSQL-backed config store with in-memory cache (5-minute TTL).
- `server/db.ts`: Drizzle ORM database connection.
- `shared/schema.ts`: Defines `aiConfigTable` and related types.
- Supports 7 agent types, each configurable with system prompt, model, max_tokens, and temperature.

**Integration:** All `OpenAIService` methods read configurations from `getAIConfig()`, allowing immediate application of changes.

**AI Analysis Caching:** Patient analysis results are cached in Directus (`ultima_analise_ia`, `data_ultima_analise`) with a 24-hour TTL, significantly reducing OpenAI API calls.

## Stripe Payment Integration
Handles Stripe webhooks with a dual-strategy user lookup to mitigate Directus API cache delays. Frontend includes manual refresh for subscription status.

## State Management
Client-side state is managed with TanStack Query for server state and React hooks for local component state. An `AuthContext` manages user authentication and data.

## User Registration Flow
A 3-step form for nutritionist registration covers basic info, professional details (including clinic WhatsApp), and AI bot configuration, saving all data to Directus `directus_users`.

## Settings & Profile Management
The `/settings` page allows nutritionists to update personal information (name, email, CPF/CNPJ, three distinct phone numbers: personal, clinic WhatsApp, bot WhatsApp, address, specialization) and customize AI agent behavior. The bot WhatsApp number is synchronized across two Directus fields for consistency.

## WhatsApp Schedule System
Automated WhatsApp message scheduling supports "Reactivation", "Meal Feedback", and "Post-Consultation" dispatch types.

**Directus Collections:**
- `whatsapp_schedules`: Stores schedule configurations.
- `whatsapp_schedule_logs`: Audit log for sent messages.

**Implementation:** `ScheduleService` handles CRUD operations via Directus API. Schedules are patient-specific and disabled by default.
